# コメント

## 基本方針

コメントはWebSocketでリアルタイム配信します。

```text
wss://stream.example.com/ws/rooms/main/comments
```

このプロダクトは配信専用です。動画アーカイブを扱わないため、コメントに動画内の再生位置は持たせません。コメントの時刻情報は投稿時刻の`sentAt`だけです。

視聴画面で流すコメントは、WebSocketで現在届いたコメントだけです。ページリロード時やルーム切り替え時に、保存済みコメント履歴を動画上で再アニメーションしてはいけません。

保存済みコメントは管理画面の確認、削除、監査用途で使います。

## メッセージ形式

```json
{
  "type": "comment",
  "roomId": "main",
  "commentId": "01HX0000000000000000000000",
  "body": "こんにちは",
  "direction": "rightToLeft",
  "color": "#ffffff",
  "fontSize": "medium",
  "sentAt": "2026-05-09T06:00:00Z"
}
```

`sentAt`はサーバーがコメント保存時に付与します。クライアントから送信させません。

## 方向

| 値 | 表示 |
| --- | --- |
| `rightToLeft` | 右から左へ流れる |
| `leftToRight` | 左から右へ流れる |
| `topToBottom` | 上から下へ流れる |
| `bottomToTop` | 下から上へ流れる |
| `fixedTop` | 上部に一定時間固定表示する |
| `fixedBottom` | 下部に一定時間固定表示する |

## バリデーション

- `body`は1文字以上100文字以下にします。
- `direction`は定義済みの値だけを許可します。
- `color`は許可色の一覧から選びます。
- `fontSize`は`small`、`medium`、`large`だけを許可します。
- 同一接続からの投稿は秒間1件までに制限します。
- 投稿者の保存仕様は`docs/backend.md`を参照します。

## 表示

フロントエンドはコメントを方向ごとにレーンへ割り当てます。同じレーンでは、前のコメントが一定距離を進むまで次のコメントを置きません。

縦方向コメントは横方向コメントと別レイヤーにします。

## 表示上限

| 画面 | 同時表示コメント上限 |
| --- | ---: |
| PC | 120件 |
| タブレット | 80件 |
| スマホ | 45件 |
